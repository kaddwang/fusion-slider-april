01 觀念覺醒

Indie Hacker

未來 3 年最被低估的職涯選擇

什麼是 Indie Hacker？

定義： 一個人或極小團隊，不依賴創投 (VC)，從 Idea 到變現全包辦。

目標： 不追求成為獨角獸後賣掉公司，快速透過產品獲得利潤 與 達到時間與收入自由。

01 觀念覺醒

這不是兩條平行的路，是完全不同的遊戲規則

維度 傳統新創 (Startup) 獨立開發者 (Indie Hacker)
資金來源 VC 募資、天使投資人 自力更生 (Bootstrap)

開發節奏 寫 BP、Pitch、開發長週期 MVP 上架、市場直接驗證
團隊規模 強調擴張、傳統組織 一人軍隊 (或 < 3 人)

03 實戰系統

從點子到營收：4 步打造你的產品事業

重點導向： 不要追求完美，先上線能真正解決付費客戶痛點的最小功能。第一週行動清單：找 3 個痛點進行訪談；用一個 Fake Door 登陸頁測需求；訂好初步定價（$/月 或 LTD）；把頁面發到相關社群收集回饋與 Email。

A. 拆解巨頭 (Unbundling)

巨頭什麼都賣，但什麼都不精。你的戰術是：不做瑞士刀，只做巨頭身上最痛的那塊肉。

🇹🇼 台灣案例：專注的力量
Yahoo 拍賣 (什麼都賣) ➡ Pinkoi (只做設計)

早期買禮物去 Yahoo 像大海撈針。Pinkoi 把「設計文創」拆解出來獨立做，就成了亞洲最大設計平台。

B. 痛點發掘 (Pain-point Mining)

不要憑空想點子，去聽聽大家在罵什麼。憤怒在哪裡，商機就在哪裡。解決那個讓人想摔手機的瞬間。

🇹🇼 台灣案例：通勤族的憤怒
公車還有多久來？(焦慮) ➡ Bus+ / 台北等公車

這款 App 沒做什麼高科技，它只解決了一個最讓人焦慮的問題：「車到底來了沒？」，就變成了通勤必備。

C. 跨界移植 (Cross-Pollination)

創新不是無中生有，而是「舊元素的新組合」。把 A 領域的標配玩法，平移到 B 領域。

🇹🇼 台灣案例：商業模式平移
軟體訂閱制 (Netflix) ➡ 7-11 隨時取 (咖啡寄杯)

7-11 把軟體業的「訂閱/預付」模式，平移到賣咖啡上。「一次買 100 杯慢慢喝」就是零售業的 SaaS 創新。

D. 販售資訊差 (Information asymmetry)

把只有專家能做的事，變成小白也能用的工具。把「貴族」的特權，變成「平民」的玩具。

🇹🇼 台灣案例：開店門檻降維
花大錢請工程師寫 Code ➡ Shopline / 91APP

以前開品牌官網門檻極高，開店平台把技術降維成「模組」，讓不懂程式的店家也能輕鬆做電商。

🧠 AI Coding

IDE / Editor

Cursor / VS Code

首選 Cursor：自帶 Claude 3.5 的編輯器，寫 Code 像在對話。

UI System

Tailwind CSS：現代切版必備。
Shadcn/ui：目前最紅的複製貼上元件庫。
v0.dev：用 AI 生成上述兩者的代碼。

🏗️ Full Stack

Framework

Next.js / React / Nuxt

Next.js：React 生態系王者，資源最多。
Nuxt：如果你更喜歡 Vue 的簡潔，這是最佳解。
選主流框架是為了讓 AI 能更準確地輔助你。

🗄️ DB

低成本BaaS必備

Supabase / Firebase

Supabase：開源版 Firebase，基於 SQL，功能更強大。
Firebase：Google 老牌產品，NoSQL 起手最快。
兩者皆含 Auth、資料庫、檔案存儲。

🚀 Deploy

Hosting

Vercel / Zeabur / GCP / AWS

Vercel：Next.js 的親爸，部署體驗最好。
Zeabur：新一代的平台，台灣人創辦的！
GCP / AWS：大廠方案，適合需要高度客製化的專案。

💰 Revenue

Payments

Stripe / 藍新 / 綠點 / LemonSqueezy

Stripe：全球支付霸主，費率低但需自理稅務。
藍新 & 綠點：台灣本地支付方案，支援多種在地付款方式，但海外信用卡刷不過。
LemonSqueezy：專為 SaaS 設計，代繳全球稅務 (MoR)，省心但費率稍高。

Step 03. 定價策略

定價即產品 (Pricing is Product)

定價不只是數學題，更是心理學：六大變現要點

1. PPP 動態定價 🌍

不要用美國的薪資水平賣給全世界。針對低收入國家自動打折，換取市佔率。

🇺🇸 美國：$29 / mo
🇮🇳 印度：$9 / mo (自動偵測 IP)
2. 錨點效應 (Decoy) ⚓

當有三個方案時，用戶往往會選擇中間那個。利用昂貴的高級版（Anchor）來襯托標準版的划算。

⚪ Basic: $9 (缺功能)
🔵 Pro: $19 (最推)
⚫ Max: $49 (襯托用)
3. LTD 早期買斷 💰

產品初期為了生存，可以賣「終身買斷制（Lifetime Deal）」快速換現金，驗證需求並獲取種子用戶。

初期：$99 終身買斷 (限 100 人)
後期：轉為月費訂閱制
4. 年繳誘因 (Cashflow) 💸

SaaS 最怕現金流斷裂。用「兩個月免費」的誘因，讓用戶一次付清一年。

月繳：$19 / mo
年繳：$190 / yr (省下 17% + 先拿現金)
5. 用量計價 (Usage) ⚡

針對 AI 類產品，成本隨使用量增加。不要吃到飽，改用 Credit/Token 模式確保利潤。

❌ 無限生成圖片
✅ 每月 500 點，用完加購
6. 老鳥紅利 (Grandfather) 👴

漲價是必然的。但承諾舊用戶「終身維持原價」，他們會成為你最死忠的推廣者。

新用戶：$29 / mo (漲價後)
老用戶：$19 / mo (永不漲價)
Indie Hacker 思維

一人公司的存活鐵律

資源有限，別為了虛榮指標浪費子彈

🚀
1. 交付為王 (Shipping)
✕ 閉門造車，堅持功能要 100% 完美
✔ 解決核心痛點就發布，讓市場教你改
尷尬是必要的成本。
產品是為了被「用」而生，不是為了被「展示」。
💰
2. 銷售思維 (Sales Mindset)
✕ 用「免費」換取大量用戶，卻沒人付錢
✔ 敢直接談錢，用「付費」來驗證需求
沒有交易，就沒有驗證。
收不到錢的「喜歡」只是客套。
🛡️
3. 尊嚴底氣 (Dignity Money)
✕ 因為缺錢恐慌，被迫接爛案子維持生存
✔ 備好 2 年存糧 (FU Money)，只做對的事
保持飢渴，但不要飢餓。
你的存款長度，決定了你堅持原則的硬度。

指示詞為何失準？

常見錯誤原因

❌ 缺乏背景

「幫我寫個網頁。」(什麼網頁？給誰看？)

❌ 假設 AI 有常識

「要做得好用一點。」(好用的定義因人而異)

❌ 步驟混亂

一次要求太多功能，導致 AI 顧此失彼。

02 Vibe Coding 與 AI 開發工具

精準心法五大重點

  • 1. 限制技術棧：明確指定 (如：Tailwind + React)。
  • 2. 檔案結構優先：先定義資料夾結構，再寫程式碼。
  • 3. 小步迭代：一次只做一個功能 (One feature at a time)。
  • 4. Review Code：不要盲目複製，檢查 AI 的邏輯。
  • 5. Don't fight the AI：如果 AI 卡住，換個問法或開新對話。

表單串接 Supabase 範例


// 初始化 Supabase Client
const supabaseUrl = 'YOUR_SUPABASE_URL';
const supabaseKey = 'YOUR_ANON_KEY';
const supabase = createClient(supabaseUrl, supabaseKey);

// 監聽表單提交
form.addEventListener('submit', async (e) => {
    e.preventDefault();
    const email = document.getElementById('email').value;

    // 寫入資料庫
    const { data, error } = await supabase
        .from('subscribers')
        .insert([ { email: email } ]);

    if (error) console.error('Error:', error);
    else alert('預約成功！');
});
